## Schema Overview

Six tables, all in the `public` schema with RLS enabled.

### `prices`
One row per metal × source × contract × timestamp. The main price feed.

| Column | Type | Notes |
|---|---|---|
| `metal` | text | `copper`, `aluminium`, `zinc`, `nickel`, `lead`, `tin`, `gold`, `silver` |
| `source` | text | `lme_official`, `yahoo`, `yahoo_fallback` |
| `contract` | text | `cash`, `3m`, `front_month` |
| `price` | numeric | |
| `currency` | text | Default `USD` |
| `unit` | text | `tonne`, `troy_oz`, `lb` |
| `as_of` | timestamptz | Trading timestamp |
| `prev_close` | numeric | Previous close (nullable) |
| `change_pct` | numeric | % change vs prev close (nullable) |

**Written by:** Edge Function `ingest-prices`
**Unique constraint:** `(metal, source, contract, as_of)`

---

### `lme_stocks`
Daily LME warehouse inventory by metal.

| Column | Type | Notes |
|---|---|---|
| `metal` | text | |
| `on_warrant` | bigint | Live, deliverable stock |
| `cancelled_warrants` | bigint | Earmarked for delivery |
| `total_stock` | bigint | `on_warrant + cancelled_warrants` |
| `cancelled_pct` | numeric | **Generated column** — `(cancelled / total) × 100` |
| `as_of` | date | |

**Written by:** Edge Function `ingest-prices`
**Unique constraint:** `(metal, as_of)`

---

### `news`
Aggregated headlines from RSS and scrapers. Sentiment fields are null until the classifier runs.

| Column | Type | Notes |
|---|---|---|
| `source` | text | Feed identifier |
| `headline` | text | |
| `url` | text | Unique |
| `summary` | text | Nullable |
| `published_at` | timestamptz | Indexed (desc) |
| `metals` | text[] | AI-populated — which metals the story covers |
| `sentiment` | text | `bullish`, `bearish`, `neutral` — AI-populated |
| `sentiment_rationale` | text | AI explanation |
| `relevant` | boolean | null = unclassified |

**Indexes:** `published_at DESC`; partial index on `relevant = true`
**Written by:** Edge Functions `ingest-news` (insert) → `classify-news` (update sentiment fields)

---

### `briefs`
Daily morning brief generated by Claude.

| Column | Type | Notes |
|---|---|---|
| `content` | text | Full Markdown brief |
| `generated_at` | timestamptz | When generated |
| `for_date` | date | Unique — one brief per trading day |

**Written by:** Edge Function `generate-brief`

---

### `arb_history`
Daily LME–COMEX copper spread for arbitrage tracking.

| Column | Type | Notes |
|---|---|---|
| `as_of` | date | Unique |
| `lme_copper_usd_tonne` | numeric | |
| `comex_copper_usd_tonne` | numeric | |
| `spread_usd` | numeric | **Generated** — `comex - lme` |
| `spread_pct` | numeric | **Generated** — `(spread / lme) × 100` |

**Written by:** Edge Function `ingest-prices`

---

### `cron_log`
Audit trail for every scheduled job run.

| Column | Type | Notes |
|---|---|---|
| `job` | text | Job name (e.g. `ingest-prices`) |
| `status` | text | `success`, `partial`, `failed` |
| `message` | text | Optional detail / error |
| `ran_at` | timestamptz | When the run finished |

**Written by:** All Edge Functions at end of each execution

## Row Level Security

RLS is **enabled on all 6 tables**.

- **Anon key** (frontend): `SELECT` only — via `"public can read <table>"` policies
- **Service role key** (Edge Functions): full access — bypasses RLS by default in Supabase, no explicit write policies needed

Never expose the service role key to the browser.

## Vault Secrets

Edge Functions retrieve them at runtime via:
```sql
select decrypted_secret from vault.decrypted_secrets where name = 'project_url';
```

## Connecting to the Database

### From Edge Functions (Deno)
```ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
)
```
Both env vars are auto-injected by the Supabase Edge Runtime — no manual config needed.

### From the Next.js frontend (anon, read-only)
```ts
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
)
```

### Running migrations
```bash
# Link project (first time only)
supabase link --project-ref tevakneehbiwltboavlw

# Push new migrations
supabase db push

# Pull remote schema changes back to local
supabase db pull
```
